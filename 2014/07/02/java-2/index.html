内部类：类中定义一个类。可以声明为public或者private。

使用static可以声明内部类，用static声明的内部类变成了外部类，但是用static声明的内部类不能访问非static的外部类属性。

也可以在方法中定义一个内部类，但是在方法中定义的内部类不能直接访问方法中的参数。如果方法中的参数要想被内部类所访问，则参数前必须加上final关键字。

面向对象－－－高级

继承： class 子类 extends 父类｛｝，只允许多层继承，而不能多重继承。继承的子类有时称为派生类。子类不能直接访问父类的私有成员，子类可以调用父类中的非私有方法，但是不能直接调用父类中的私有成员。

子类对象的init过程，子类对象在init之前必须先调用父类的构造方法后再调用子类自己的构造方法。

方法的覆写：子类定义了与父类同名的方法，但是在方法覆写时必须考虑到权限，被子类覆写的方法不能拥有比父类方法更加严格的访问权限。

不过子类为什么要定义跟父类名称一样的方法呢？

private<default<public 如果父类中方法时public 子类必须也是public，否则程序无法编译。 子类权限只能放大。

private方法不能覆写。

如果想要继续调用父类中被覆写过的方法，使用super关键字就可以。

final关键字

  • 使用final声明的类不能有子类

  • 使用final声明的方法不能被子类所覆写

  • 使用final声明的变量既成为常量，常量不可以修改

抽象类

java中可以创建一种类专门用来当作父类，这种类为“抽象类”，抽象类的作用类型模板，其目的是要设计者依据它的格式来修改并创建新的类，但是并不能直接由抽象类创建对象，只能通过抽象类派生出新的类，再由它来创建对象。

  • 包含一个抽象方法的类必须是抽象类

  • 抽象类和抽象方法都要使用abstract关键字声明

  • 抽象方法只需声明而不需要实现

  • 抽象类必须被子继承，子类（如果不是抽象类）必须覆写抽象类中的全部抽象方法

  • 抽象方法不要使用private声明
